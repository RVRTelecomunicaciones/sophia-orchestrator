## Promotion gate (Day 7, 2026-05-15)

Promotion to `v0.2.0` is permitted ONLY when ALL conditions are met:

- [ ] Every cell in the matrix above is 🟢 GREEN (no 🟡 carried
  over without a "resolved" note; no 🔴 anywhere).
- [ ] `shasum -a 256 docs/specs/sophia-wire-v1.md` is identical
  across both repos AT THE PROMOTION COMMIT.
- [ ] `make test` + `make contract` (cli) and `make test-unit` (orch)
  pass on the to-be-promoted commits.
- [ ] No issue raised during the soak references the v0.2.0 wire
  surface and remains open without an explicit "won't fix /
  scheduled for v0.3.0" decision.
- [ ] Operator smoke `run → detach → attach` succeeded at least
  once during the window against a real (non-synthetic)
  orchestrator.

If ALL met → tag promotion (run the commands in §"Promotion
procedure" below). Both repos tag on the SAME day.

If ANY condition fails → triage:

1. Open an issue per RED cell with reproducer.
2. Fix on the branch the issue lives in (`m10/*` for cross-cutting,
   `<repo>/main` for repo-local).
3. Re-cut `v0.2.0-rc.2` (or higher) on the affected repo(s).
   Coordinated rc bumps preferred — bump both even if only one
   needs the fix, so soak-window tracking stays simple.
4. Restart the 7-day soak window from rc.N day-0.

## Promotion procedure (Day 7, conditional)

The `scripts/promote-v0.2.0.sh` script automates the closing smoke pass
+ all six release-blocker gates and (only with `--confirm`) creates the
annotated `v0.2.0` tags on both repos.

```bash
# Dry-run: runs every gate, reports RED/GREEN, creates no tags. Safe to
# run any day during the soak window.
./scripts/promote-v0.2.0.sh

# Day 7, all gates green: actually create the tags (still no remote push).
./scripts/promote-v0.2.0.sh --confirm

# After explicit sign-off: push the tags. The cli push triggers
# goreleaser → public v0.2.0 release artifacts.
( cd sophia-cli            && git push origin v0.2.0 )
( cd sophia-orchestator    && git push origin v0.2.0 )
```

The script asserts:
1. SHA256 cross-repo (D-M10-16 #1)
2. cli `go test ./...` + `GOWORK=off go test ./...` + `make contract` (D-M10-15 + D-M10-16 #2)
3. orch `go test ./...` + `GOWORK=off go test ./...` (D-M10-15)
4. CHANGELOG `### Compatibility` section in both repos (D-M10-16 #4)
5. Soak matrix: zero open RED entries (D-M10-16 #3)
6. No pre-existing local `v0.2.0` tag on either side (no double-tag)

Exit 1 on any RED gate; exit 0 only when all six are green. The
`v0.2.0` tags MUST land on the same calendar day per D-M10-11.

## Resolved during Day 0 (2026-05-08, post-rc.1 push)

- **cli CI workflow `gofmt` + `gosec` failures** on commit `2c10857`.
  - **Symptom:** `gofmt` flagged the `pkg/contract/{events,routes,types}.go`
    files for tab alignment; `gosec` G101 flagged `HeaderAPIKey`,
    `HeaderAPIKeyLegacy`, and `EnvAPIKey` constants as "potential
    hardcoded credentials" (false positive — they are HTTP header /
    env var NAMES, not values).
  - **Fix:** `gofmt -w` on the three contract files; `// #nosec G101`
    inline annotations on the three constants explaining the false
    positive (matches existing repo convention from
    `composeexec.writeFile0644`).
  - **Validation:** `gosec -severity high -quiet ./...` clean; 21
    cli packages still pass; `go test -tags=contract` still green;
    SHA256 cross-repo invariant unchanged
    (`097be339...d19fd6c5`).
  - **Tag impact:** the `v0.2.0-rc.1` tag at `f6caae3` is
    untouched. The fix lands in a follow-up commit on `main` that
    is part of the v0.2.0-rc.1 → v0.2.0 soak window; if soak closes
    cleanly, `v0.2.0` final will tag the post-fix commit, not the
    rc.1 commit.

    `.golangci.yaml` to v2 schema and bump
    `golangci-lint-action@v6` → `@v7` with `version: v2.12`
    (matches the cli's working setup). Substantive — out of Phase 7
    scope.
  - **Soak impact:** does NOT block v0.2.0 promotion per the
    matrix's promotion gate; tracked here for visibility only.
